## Module Details

### 1 The Improved LSTM Model

> Full documentation: `Data&Model/1 The improved LSTM model/README-improved lstm.md`

This is the core deep learning module implementing a multi-source information fusion LSTM with attention mechanism for Bitcoin price direction prediction.

#### Submodules

##### train/
Implements the complete training pipeline. Run `python main.py` to start training with multi-source BTC factor features from `train/dataset/dataset/`. Model checkpoints (`.ckpt`) are saved to `train/ckpt/`. Dataset splits are controlled by the `Group` parameter in `config.py` (7 time-based groups from 2017–2022).

##### test/
Model evaluation and strategy generation. Run in order:
1. `python test.py` — evaluates across all Groups, outputs `test_metrics.csv` and confusion matrices
2. `python strategy.py` — generates trading signals from predictions (supports adjustable threshold)

##### Backtesting system/
Quantitative backtesting framework. Run `python backtest.py` to evaluate strategies with performance metrics (max drawdown, Sharpe ratio, cumulative returns). Supports both zero-cost and transaction-cost modes (toggle via `trade_loop_back1()` / `trade_loop_back2()` around line ~301).

##### ablation_experiment/
Ablation study isolating four BTC factor groups:
- `input_size_1` — Blockchain
- `input_size_2` — Market
- `input_size_3` — Macroeconomic Environment
- `input_size_4` — Public Attention

Run `python All_inference.py` to reproduce all ablation results. For the LSTM-without-attention baseline, run `cd "LSTM (without attention mechanism)" && python test.py`.

##### Different Fusion Methods/
Comparison of alternative fusion strategies (Fusion Method I and II). Run `python test_all.py` in each method's directory. Results are consolidated in `test_results_all/summary.csv`.

##### ML/
Traditional machine learning baselines. Navigate to `final_code/ML/ml/` and run `python main.py`. Accuracy statistics are saved to `TEST_result.txt`.

#### Recommended Reproduction Order
1. `test/test.py` → obtain prediction results and metrics
2. `test/strategy.py` → generate trading strategies (default & 0.6 threshold for all 7 Groups)
3. `Backtesting system/backtest.py` → backtest (zero-cost first, then with transaction costs)

The resulting return curves and performance metrics constitute the core experimental results.

### 2 Time Series Model

> Full documentation: `Data&Model/2 Time series model/README-time series.md`

Forecasts Bitcoin daily returns using 10 classical time series models
(AR/ARMA/ARCH/GARCH/EGARCH) and constructs two buy-and-hold trading strategies
with full backtesting.

**Environment:** conda 4.13.0 (separate from LSTM environment)
**Dependencies:** pandas, numpy, scipy, statsmodels, arch, matplotlib, seaborn, scikit-learn

#### Code Structure (3-step pipeline)

| Step | Script | Purpose |
|:----:|------|------|
| 1 | `code/01_prediction_accuracy.py` | Evaluate 10 models across 5 periods x 4 windows |
| 2a | `code/02_strategy_no_threshold.py` | Generate no-threshold trading signals |
| 2b | `code/02_strategy_threshold.py` | Generate threshold trading signals |
| 3 | `code/backtesting/backtest.py` | Backtest strategies (with/without transaction fee) |

#### Models (10 total)

AR(1), AR(2), ARMA(1,1), ARMA(2,2), AR(1)-ARCH, AR(2)-ARCH,
AR(1)-GARCH, AR(2)-GARCH, AR(1)-EGARCH, AR(2)-EGARCH

EGARCH excluded from strategies/backtesting due to poor directional accuracy.

#### Sample Periods (5)

| Period | Start | End |
|--------|-------|-----|
| Baseline | 2024-04-01 | 2024-09-30 |
| Event 1 before | 2019-07-29 | 2020-01-29 |
| Event 1 after | 2020-01-30 | 2020-07-30 |
| Event 2 before | 2021-08-24 | 2022-02-23 |
| Event 2 after | 2022-02-24 | 2022-08-24 |

Rolling windows: 18, 30, 60, 90 trading days.

#### Input Data

- `data/raw/btc.csv` — Bitcoin full price history
- `data/raw/suoyin.csv` — Benchmark index data
- `data/windows/` — 20 pre-split CSV files (5 periods x 4 windows)

#### Output

- `outputs/accuracy/MAPE-RMSE-Accuracy-Results.docx` — Consolidated accuracy records
- `outputs/strategies/` — 80 strategy signal files (.txt)
- `outputs/backtest/backtest_results.csv` — Backtest results
- `outputs/figures/` — 80 forecast charts (.png)

#### Usage

Each script requires modifying two lines before running:
- **Line 35**: data path -> target window CSV
- **Line 91**: `end_loc` -> window size

Strategy scripts overwrite output on each run; rename output files with period suffix
(baseline / beforeevent1 / afterevent1 / beforeevent2 / afterevent2) after each execution.

Backtest must be run from `code/backtesting/` directory and requires strategy files
placed in `code/backtesting/strategies/`. Two modes: `trade_loop_back2` (no fee)
and `trade_loop_back1` (with fee) — toggle via lines 371-372 and 403-404.

### 3 Asset Pricing Model

> Full documentation: `Data&Model/3 Asset pricing model/README-asset pricing.md`

Empirical asset pricing analysis of Bitcoin daily returns, implemented in **Stata**. Tests Bitcoin's exposure to traditional factor models (Fama-French), currencies, and commodities.

**Requirements:** Stata with `outreg2`, `jb6`, `mergemany` packages.

**Usage:** Set working directory to this folder in Stata and run `do "code/Asset pricing model.do"`. Outputs appear in `result/`.
